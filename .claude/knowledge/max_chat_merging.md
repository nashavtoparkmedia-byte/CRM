# Интеграция MAX Web Scraper с CRM

## Проблема Svelte UI в MAX
Интерфейс веб-версии MAX построен на Svelte и имеет специфичное поведение:
Когда поступает входящее сообщение от контакта, который **не сохранен в справочнике MAX** или для которого MAX **не отображает номер телефона в UI**, в DOM-дереве отсутствует физическая строка с номером телефона (вида `+79...` или `922...`).

В результате скрапер `maxBrowser.js` не может "вытащить" телефон из верстки. Он извлекает только Имя контакта (например, `Александр`).

## Появление "Ghost Chats" (Анонимных чатов)
Когда webhook в CRM (`src/app/api/webhook/max/route.ts`) получает данные от скрапера с `phone="Александр"`, он пытается найти `Driver` по совпадению `fullName`.
Поскольку имя `Александр` очень распространено (в БД может быть "Александр Иванов", "Александр Ремезов" и т.д.), CRM не может однозначно определить, какому водителю принадлежит сообщение.

**До исправления**: В случае неоднозначного имени webhook создавал полностью новый чат `max_name:Александр` без привязки к `driverId` (т.н. "ghost chat"). Это приводило к разрыву диалога — менеджер писал в "Ремезов Александр", а ответ приходил в пустой анонимный чат "Александр".

## Текущее решение (Fix in `route.ts`)
Чтобы решить эту проблему, в webhook (`src/app/api/webhook/max/route.ts`) была внедрена логика **неявного связывания через фокус недавней активности**:

1. Если scraper присылает короткое не-телефонное имя (`phoneDigits.length < 10`), система сначала пытается сделать точный поиск по имени в таблице `Driver`.
2. Если `matchedDriverId` не найден (имя двусмысленное), система ищет **недавно активные чаты в канале MAX**, которые *уже привязаны к конкретным водителям*.
3. Если среди 20 последних чатов находится чат, принадлежащий водителю, чье `fullName` содержит искомое слово (например, у водителя "Ремезов Александр" есть активный MAX-чат, а скрапер прислал "Александр"), система делает вывод, что это ответ в рамках текущего диалога.
4. Система извлекает *реальный* телефонный номер водителя из базы (в данном случае `+79222155750`) и подменяет им сырое значение `phone` из скрапера.
5. После этого весь флоу идет штатно: сообщение прикрепляется к правильному `externalChatId` (вида `max:79222155750`).

### Код логики:
```typescript
const searchName = (driverName || phone).trim();
const recentDriverChats = await prisma.chat.findMany({
    where: { channel: 'max', driverId: { not: null } },
    include: { driver: true },
    orderBy: { lastMessageAt: 'desc' },
    take: 20
});

const matchedActiveChat = recentDriverChats.find(c => 
    c.driver && c.driver.fullName.toLowerCase().includes(searchName.toLowerCase())
);

if (matchedActiveChat && matchedActiveChat.driver.phone) {
    phoneDigits = matchedActiveChat.driver.phone.replace(/\D/g, '');
    // Найдено совпадение! Подменяем ID чата реальным номером.
}
```

## Как тестировать, если снова сломается:
1. Использовать `browser_subagent` для проверки DOM в MAX Web, чтобы понять, отдает ли он телефон.
2. Использовать скрипт `test-max-webhook.js` в папке `gravity-mvp`, отправив POST запрос в `/api/webhook/max` с `phone="Александр"` и `driverName="Александр"`.
3. Убедиться через скрипт `verify-max-webhook.ts`, что новое сообщение "приклеилось" к чату водителя, а не создало ghost chat.
