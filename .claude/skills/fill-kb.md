# Fill KB — Наполнение базы знаний AI

## Контекст
KnowledgeBaseEntry используется IntentClassifier для определения намерения водителя. Без записей AI даёт низкий confidence и всё эскалирует.

## Что делать

### 1. Показать текущее состояние KB
```sql
SELECT id, category, title, "isActive", "createdAt" FROM "KnowledgeBaseEntry" ORDER BY category
```
Если пусто — сказать "KB пуста, давай наполним".

### 2. Спросить пользователя что добавить
Предложить стандартные категории для таксопарка:
- **payments** — вопросы о выплатах, зарплате, бонусах
- **schedule** — расписание, смены, выходные
- **car_issues** — проблемы с машиной, ТО, ремонт
- **documents** — документы, справки, договоры
- **rules** — правила парка, штрафы, регламенты
- **general** — общие вопросы, приветствия

Спросить: "Какие категории добавить? Или опиши свои."

### 3. Для каждой категории создать записи
Формат записи KnowledgeBaseEntry:
```sql
INSERT INTO "KnowledgeBaseEntry" (id, category, title, content, keywords, "isActive", "createdAt", "updatedAt")
VALUES (gen_random_uuid(), $category, $title, $content, $keywords, true, NOW(), NOW())
```

Где:
- **title** — краткое название (напр. "Когда выплата")
- **content** — полный ответ который AI может использовать
- **keywords** — массив ключевых слов для matching (JSON array)
- **category** — категория из списка выше

### 4. Валидация
После добавления:
- Показать все записи в таблице
- Проверить что keywords не пересекаются между категориями
- Предложить тестовый прогон: "Отправить тестовое сообщение чтобы проверить?"

### 5. Тестовый прогон (опционально)
Если пользователь согласен — использовать логику из /test-pipeline с тестовым сообщением, релевантным добавленной KB записи.

## Важно
- Всегда спрашивать пользователя про содержание ответов — не выдумывать
- Ответы должны быть на русском языке
- Keywords должны включать типичные формулировки водителей (разговорный стиль)
