# Plan First — Сначала план, потом код

## Правило
При получении задачи НЕ начинать реализацию. Сначала:
1. Показать как понял задачу
2. Задать вопросы если есть неясности
3. Предложить план реализации
4. Ждать подтверждения

## Алгоритм

### 1. Как понял задачу
Переформулировать задачу своими словами в 1-3 предложениях. Цель — показать что понял правильно. Не пересказывать дословно, а показать суть.

### 2. Уточняющие вопросы (если нужны)
Спросить только то, без чего нельзя начать. Не задавать вопросы ради вопросов. Примеры когда нужно:
- Неясно где именно в UI разместить элемент
- Несколько вариантов реализации с разными trade-offs
- Задача затрагивает несколько модулей — уточнить скоуп
- Непонятно поведение в edge cases

### 3. План реализации

Написать план в виде пронумерованных шагов. Для каждого шага указать файл и что именно делаю. Обязательно включить:

**Основной план (Plan A):**
```
1. <шаг> — <файл> — <что делаю>
2. <шаг> — <файл> — <что делаю>
...
```

**Запасной план (Plan B) — если основной не сработает:**
```
Если <что может пойти не так>:
  → <альтернативный подход>
```

Примеры запасных планов:
- "Если Prisma generate упадёт с EPERM → использую $executeRaw"
- "Если API endpoint не отвечает → проверю через прямой запрос к БД"
- "Если компонент конфликтует с существующим layout → вынесу в отдельную страницу"

**Риски:**
- Что может сломаться от этих изменений
- Какие модули затронуты

**Проверка:**
- Как буду тестировать (build / скрипт / визуально / pipeline тест)
- Какие edge cases проверю

### 4. Ждать подтверждения
Закончить фразой типа:
> Если ок — скажи "в реализацию" и я начну.

Не начинать пока не получу одно из:
- "в реализацию"
- "го"
- "делай"
- "ок"
- "да"
- "начинай"
- любое явное подтверждение

### 5. Если пользователь поправил
- Обновить план с учётом правок
- Показать обновлённый план
- Снова ждать подтверждения

## Когда НЕ применять
- Пользователь написал **"Full Auto"** — сразу в реализацию
- Задача тривиальная и однозначная ("поправь опечатку в строке 15") — можно сразу делать
- Пользователь явно сказал "без вопросов, просто сделай"

## Формат ответа

```
**Как понял:** <суть задачи>

**Вопросы:**
- <вопрос 1>
- <вопрос 2>
(или "Вопросов нет, задача понятна")

**План A:**
1. <шаг> — <файл> — <что делаю>
2. <шаг> — <файл> — <что делаю>
3. ...

**План B (если A не сработает):**
- Если <проблема> → <альтернатива>

**Риски:** <что может сломаться>

**Проверка:** <как протестирую>

Если ок — скажи "в реализацию".
```
